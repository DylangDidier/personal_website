Haddy Handheld Defect Logger

Edge ComputingIoT AutomationHardware Integration
Overview

Engineered a custom, 3D-printed smart camera and data logging device to optimize quality control (QC) workflows on the manufacturing shop floor. Bypassed tedious, multi-tool manual tracking systems by consolidating laser targeting, distance telemetry, high-resolution image processing, and secure cloud synchronization into a single-click, ruggedized point-and-shoot tool.

My Contributions
  • 3D Design & Modeling: Modeled a custom, multi-part ergonomic enclosure from scratch using CAD software, meticulously placing internal standoffs, mounting brackets, and tolerances to securely house the Raspberry Pi, touchscreen, ToF sensor, battery dock, and wiring harness.
  • Hardware & Systems Architecture: Wired and programmed an IMX708 high-resolution camera, physical tactile trigger button, 5V targeting laser, and a VL53L0X Time-of-Flight (ToF) distance sensor to communicate over I2C and GPIO pins via a Raspberry Pi 4 brain.
  • Telemetry Math & Automation: Developed trigonometric logic parsing real-time ToF distance inputs to dynamically calculate the camera’s active field of view in millimeters, enabling human-error-free defect scale evaluations.
  • Image Processing Pipeline: Formulated an automated image manipulation script using Python (Pillow) to dynamically draw bounding grids and permanently stamp a white metadata footer onto captured frames, embedding timestamps, material configurations, and calculated metrics directly into the JPEG buffers.
  • Timing & Race Condition Isolation: Resolved critical rolling-shutter artifacts and red glare saturation by programming tight hardware micro-delays, shutting off the laser diode exactly 0.25 seconds prior to camera shutter release.
  • Industrial Power Engineering: Designed an isolated power system running completely off a standard 18V/20V DeWalt power tool battery using a step-down buck converter, allowing hot-swappable floor operations.
  • UI Development & Deployment: Authored a dark-mode touch user interface using CustomTkinter for quick operator categorization inputs and configured a headless Linux boot sequence with failsafe Bash scripts to launch the application into fullscreen mode upon boot.
Results
  • Process Consolidation: Replaced a fragmented clipboard-and-camera pipeline with an immediate enterprise-grade digital capture system, reducing QC processing times drastically.
  • Instant Pipeline Visibility: Integrated Rclone to handle headless cloud uploading, feeding live CSV telemetry sheets and documented logs straight to a secure engineering directory seconds after on-floor triggers.
  • Cost Optimization: Delivered a fully serviceable, rugged smart scanner using off-the-shelf maker components and 3D-printed enclosures, minimizing maintenance costs relative to proprietary alternatives.

Microfactory Training & Knowledge Hub

Web DevelopmentTechnical WritingJira API
Overview

Built an internal training website to help factory operators learn the ropes and keep track of complex 3D printing settings in one place. The goal was to take all the scattered documents and unwritten rules about our robotic cells and put them into a single, easy-to-search platform right on the shop floor.

My Contributions
  • Internal Web Platform: Built a Markdown-based documentation site hosted locally on Linux factory machines so technicians could instantly pull up guides while working.
  • Jira API Integration: Connected shop-floor machine data and hardware calibration tickets directly to the engineering team's Jira sprints using automated API calls.
  • Operator Training System: Created a three-level training guide that taught operators how to handle everything from basic PETG prints to tricky ABS/PC blends and 45-degree multi-axis toolpaths.
  • Hardware Setup Guides: Put together a master tooling chart mapping out exact layer heights and flow rates for ten different nozzle sizes (4mm to 15mm) to stop prints from failing due to bad settings.
  • Step-by-Step SOPs: Wrote clear operating procedures for the hardware, including a strict 9-step process for vacuuming, purging lines, and setting heat profiles on the material dryers.
Results
  • Gave the factory floor team the exact resources they needed to run jobs independently, without constantly having to pull an engineer away from their desk.
  • Cut down on failed prints and layer delamination because operators finally had a quick way to double-check material settings before hitting start.
  • Sped up the onboarding process for new hires and noticeably reduced machine downtime when swapping between different plastics.
